# asdm-req-context-update: 更新需求分析上下文

## 目的

当系统发生变更时，更新对应的需求分析上下文。

## 更新场景

| 场景 | 更新内容 |
|------|----------|
| 新增业务域 | 添加新域的全维度上下文 |
| 业务能力变更 | 更新领域 context.md 中的相关维度 |
| 流程变更 | 更新 context.md 中的流程部分 |
| 数据模型变更 | 更新 context.md 中的数据模型部分 |
| 规则变更 | 更新 context.md 中的规则部分 |

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 领域名 | String | 否 | 要更新的领域名称，不填则更新全局索引 |

## 更新步骤

### 步骤 1: 识别变更内容

1. 了解发生了什么变更
2. 确定变更涉及的领域和维度

### 步骤 2: 分析变更影响

- 变更涉及哪些业务域
- 变更影响哪些维度
- 是否需要新增内容

### 步骤 3: 更新上下文

1. 读取现有的领域 context.md
2. 定位需要更新的部分
3. 更新受影响的维度内容
4. 确保更新的内容与代码一致

### 步骤 4: 标注变更

在更新时添加变更标注，便于追溯：

```markdown
<!-- 变更记录 -->
<!-- 更新时间：2026-04-24 -->
<!-- 变更原因：[原因] -->
<!-- 变更内容：[简述] -->
```

## 使用方法

```bash
# 更新指定领域的上下文
/asdm-req-context-update 用户域

# 更新全局索引（当领域列表变化时）
/asdm-req-context-update
```

## 注意事项

1. **及时更新**：代码变更后应及时更新上下文
2. **保持一致**：确保上下文与代码描述一致
3. **标注来源**：说明信息来自哪个代码位置
4. **控制大小**：更新后确保 context.md < 10KB
